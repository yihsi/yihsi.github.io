注意
OpenStack 服务运行至少需要以下组件：

  • Nova (compute, api, scheduler, and network)
  • Glance
  • Keystone
  • Neutron (除非使用了 nova-network)

Horizon 提供了对其他服务的选择性支持。参见 :ref:’system-requirements-label’ 查看支持的服务。如果配置了针对某个服务的 Keystone 节点，Horizon 可以检测到并自动启用 Keystone 提供的支持。

编辑 Horizon 的源代码

尽管当运行 stack.sh 的时候，DevStack 安装并配置了一份 Horizon，但在运行 DevStack 的服务器／虚拟机上，依照上面的说明来进行更好的开发设置。持有 Horizon 仓库的一份单独的拷贝，而不是编辑随 devstack 安装的那份拷贝，有以下几个好处：

当运行 unstack.sh/stack.sh 时，源代码没有那么容易丢失
用来开发的服务器可以在运行时相应代码的更改（除了由于压缩和编译的 JavaScript 和 CSS）
日志信息和打印语句直接在控制台显示
交互式地使用 pdb 来调试变得更容易

注意
JavaScript 和 CSS 的更改需要开发服务器的重启。并且，为了刷新非缓存的 JavaScript 和 CSS 需要在浏览器强制刷新（比如使用 Shift-F5）页面。Horizon 中的默认设置是在服务器启动的时候进行编译和压缩。如果你配置了本地拷贝进行离线压缩，那么将需要更多的步骤。

Horizon 结构

这个项目与其他的 OpenStack 项目有一点儿不同，即它下面有两个非常不同的组件：horizon 和 openstack_dashboard 。

horizon 目录包含可以用在任何 Django 项目中的通用库和组件。

openstack_dashboard 包含了使用 horizon 的参考 Django 项目。

对于开发来说，上述两个组件共享（默认）由 tools/install_venv.py 脚本创建的环境。该脚本创建一个虚拟环境并安装所有需要的包。

如果在 horizon 和 openstack_dashboard 中的任意一个中添加了依赖，那么依赖就应该被添加到 requirements.txt 。
